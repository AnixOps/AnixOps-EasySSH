### 【SSH 密钥认证】(0:25 - 4:30)

**[章节标题卡]**
"SSH 密钥认证详解"

**[画面]**

#### 原理介绍 (0:25 - 1:15)

1. **密码认证的问题** (0:25 - 0:45)
   - 展示密码输入场景
   - 说明安全风险
   - 对比图表：密码 vs 密钥

2. **密钥认证原理** (0:45 - 1:15)
   - 动画演示公私钥配对
   - 密钥生成过程
   - 认证流程图

**[旁白]**
"首先理解为什么要使用密钥认证。密码认证存在暴力破解风险，而且每次都要输入。密钥认证使用数学原理，2048位以上的加密强度，几乎不可能破解。"

"密钥认证使用一对密钥：私钥留在本地，公钥放在服务器。私钥签名，公钥验证，保证身份。"

---

#### 生成密钥对 (1:15 - 2:30)

1. **检查现有密钥** (1:15 - 1:35)
   ```bash
   ls -la ~/.ssh/
   ```
   - 展示可能的结果
   - 说明 id_rsa, id_ed25519 等文件

2. **生成新密钥** (1:35 - 2:30)
   ```bash
   # 推荐算法
   ssh-keygen -t ed25519 -C "your@email.com"

   # 或 RSA
   ssh-keygen -t rsa -b 4096 -C "your@email.com"
   ```
   - 演示完整交互过程
   - 解释密码短语的作用
   - 展示生成成功输出

**[旁白]**
"打开终端，首先检查是否已有密钥。如果看到 id_rsa 或 id_ed25519 文件，说明已有密钥，可以跳过生成步骤。"

"使用 ssh-keygen 命令生成密钥。推荐使用 ed25519 算法，更安全高效。交互过程中会询问保存位置和密码短语。密码短语是额外的安全层，即使私钥被盗，没有密码也无法使用。"

---

#### 部署公钥 (2:30 - 3:30)

1. **使用 ssh-copy-id** (2:30 - 3:00)
   ```bash
   ssh-copy-id -i ~/.ssh/id_ed25519.pub root@192.168.1.100
   ```
   - 输入服务器密码
   - 显示成功提示

2. **验证免密登录** (3:00 - 3:30)
   ```bash
   ssh root@192.168.1.100
   ```
   - 无需输入密码
   - 直接登录成功

**[旁白]**
"生成密钥后，需要将公钥复制到服务器。使用 ssh-copy-id 命令最简单。输入服务器密码一次，以后就可以免密登录了。"

"验证一下，再次 SSH 连接，应该直接进入服务器，无需输入密码。如果提示输入密码短语，说明你设置了密码短语，这是正常的。"

---

#### 配置 EasySSH Lite (3:30 - 4:30)

1. **编辑服务器** (3:30 - 3:50)
   - 右键服务器 → 编辑
   - 进入认证方式页面

2. **选择密钥认证** (3:50 - 4:15)
   - 选择 "SSH Agent"
   - 显示检测到的密钥
   - 或选择 "SSH 密钥" 手动指定
   - 输入密码短语（如果有）
   - 保存

3. **测试连接** (4:15 - 4:30)
   - 双击连接
   - 展示成功连接

**[旁白]**
"回到 EasySSH Lite，编辑服务器配置，将认证方式改为 SSH Agent 或直接指定密钥文件。"

"SSH Agent 是最方便的，它会自动使用系统中已添加的密钥。如果选择直接指定密钥，需要输入密钥路径和密码短语。"

"保存后测试连接，应该能够正常免密登录。"

### 【SSH Agent 管理】(4:30 - 6:00)

**[章节标题卡]**
"SSH Agent 使用"

**[画面]**

#### 启动 Agent (4:30 - 5:00)

1. **macOS/Linux** (4:30 - 4:45)
   ```bash
   eval "$(ssh-agent -s)"
   ssh-add ~/.ssh/id_ed25519
   ```

2. **Windows** (4:45 - 5:00)
   ```powershell
   # 管理员权限
   Get-Service ssh-agent | Set-Service -StartupType Automatic
   Start-Service ssh-agent
   ssh-add $env:USERPROFILE\.ssh\id_ed25519
   ```

**[旁白]**
"SSH Agent 是一个后台服务，临时保存你的密钥。这样你只需要输入一次密码短语，后续连接都自动使用。"

"macOS 和 Linux 用户使用 eval 命令启动 agent，然后用 ssh-add 添加密钥。Windows 10 和 11 内置了 OpenSSH，需要先在服务中启用 ssh-agent。"

---

#### 管理多个密钥 (5:00 - 6:00)

1. **添加多个密钥** (5:00 - 5:25)
   ```bash
   ssh-add ~/.ssh/id_ed25519
   ssh-add ~/.ssh/id_rsa_work
   ssh-add ~/.ssh/id_rsa_personal
   ```

2. **查看已添加密钥** (5:25 - 5:40)
   ```bash
   ssh-add -l
   ```
   - 显示所有密钥指纹

3. **在 EasySSH Lite 中使用** (5:40 - 6:00)
   - 编辑服务器 → 选择 SSH Agent
   - 显示已添加的密钥列表
   - 应用自动选择合适的密钥

**[旁白]**
"如果你有多个密钥用于不同服务器，可以都添加到 agent。使用 ssh-add -l 查看已添加的密钥。"

"在 EasySSH Lite 中选择 SSH Agent 后，应用会自动检测可用的密钥，连接到不同服务器时会自动选择正确的密钥。"

### 【批量导入配置】(6:00 - 8:30)

**[章节标题卡]**
"从 SSH Config 导入"

**[画面]**

#### 准备配置文件 (6:00 - 6:45)

1. **查看现有配置** (6:00 - 6:30)
   ```bash
   cat ~/.ssh/config
   ```
   - 展示典型配置内容
   - 解释 Host, HostName, User 等字段

2. **配置格式说明** (6:30 - 6:45)
   ```
   Host web-server
       HostName 192.168.1.10
       User admin
       Port 22
       IdentityFile ~/.ssh/id_rsa
   ```

**[旁白]**
"如果你已经维护着 ~/.ssh/config 文件，里面有大量服务器配置，EasySSH Lite 支持一键导入。"

"SSH config 文件使用简单的格式：Host 是别名，HostName 是实际地址，User 是用户名，Port 是端口，IdentityFile 指定密钥。"

---

#### 执行导入 (6:45 - 7:45)

1. **打开导入功能** (6:45 - 7:00)
   - 设置 → 导入/导出
   - 点击 "从 SSH Config 导入"

2. **选择文件** (7:00 - 7:15)
   - 自动检测到 ~/.ssh/config
   - 或手动选择其他位置

3. **预览内容** (7:15 - 7:35)
   - 显示解析到的服务器列表
   - 显示每个服务器的配置
   - 勾选/取消选择项

4. **分组策略** (7:35 - 7:45)
   - 选择分组方式
   - 根据名称自动推断
   - 创建新分组

**[旁白]**
"在设置中打开导入功能，EasySSH Lite 会自动检测默认位置的配置文件。"

"预览页面显示所有解析到的服务器，你可以取消勾选不想导入的项。还可以选择分组策略，让应用自动根据服务器名称推断分组。"

---

#### 验证导入结果 (7:45 - 8:30)

1. **查看导入结果** (7:45 - 8:00)
   - 显示导入完成统计
   - 成功、失败、跳过的数量

2. **主界面检查** (8:00 - 8:15)
   - 新服务器出现在相应分组
   - 配置完整保留

3. **测试连接** (8:15 - 8:30)
   - 双击导入的服务器
   - 验证连接正常

**[旁白]**
"导入完成后，检查统计信息。回到主界面，你会看到所有服务器已按分组整理好。"

"最后测试几台服务器，确认配置正确导入，特别是密钥认证的路径是否正确。"

### 【高级搜索技巧】(8:30 - 10:00)

**[章节标题卡]**
"高级搜索与过滤"

**[画面]**

#### 搜索语法 (8:30 - 9:15)

1. **基础搜索** (8:30 - 8:50)
   - 输入 "web"
   - 显示名称包含 web 的服务器
   - 输入 "192.168"
   - 显示该网段的服务器

2. **字段搜索** (8:50 - 9:15)
   | 语法 | 效果 |
   |------|------|
   | `host:192.168` | 按主机筛选 |
   | `user:root` | 按用户筛选 |
   | `group:prod` | 按分组筛选 |
   | `status:offline` | 按状态筛选 |

**[旁白]**
"当服务器数量很多时，搜索功能非常重要。基础搜索会匹配名称、主机、用户名。"

"更高级的是字段搜索语法。输入 host: 加上 IP 片段，只匹配主机地址。user:root 只显示 root 用户的服务器。group:prod 只显示生产环境分组。status:offline 快速找到离线服务器。"

---

2. **排除搜索** (9:35 - 9:50)
   ```

   ```

3. **保存搜索** (9:50 - 10:00)
   - 常用搜索保存为快捷方式
   - 点击快速应用

**[旁白]**
