# 教程 01：添加第一台服务器

> 预计时间：5分钟  |  难度：入门

## 目标

完成本教程后，你将学会：
- 启动 EasySSH Lite
- 添加并配置第一台服务器
- 使用密码认证建立连接
- 验证连接是否成功

---

## 准备工作

在开始之前，请确保：

1. **已安装 EasySSH Lite**
   - Windows: 下载 `.exe` 安装程序并运行
   - macOS: 下载 `.dmg` 并拖拽到 Applications
   - Linux: 下载 AppImage 或 `.deb`/`.rpm` 包

2. **拥有一台可访问的 SSH 服务器**
   - 可以是云服务器（阿里云、腾讯云、AWS、Azure等）
   - 可以是本地虚拟机
   - 或者局域网内的其他设备

3. **准备好连接信息**
   - 服务器 IP 地址或域名
   - 用户名
   - 密码或私钥

---

## 步骤详解

### 步骤 1：启动 EasySSH Lite

**Windows:**
```
双击桌面上的 EasySSH Lite 图标
或在开始菜单中搜索 "EasySSH Lite"
```

**macOS:**
```
在 Applications 文件夹中找到 EasySSH Lite
双击启动，若提示"无法验证开发者"，请前往
系统设置 > 隐私与安全性 > 安全性，点击"仍要打开"
```

**Linux:**
```bash
# 如果使用 AppImage
chmod +x EasySSH-Lite-*.AppImage
./EasySSH-Lite-*.AppImage

# 如果使用 deb/rpm
# 安装后会自动添加到应用菜单
```

**填写示例：**

| 端口 | 22 | 保持默认，除非服务器使用其他端口 |
| 用户名 | root | 你的登录用户名 |

### 步骤 6：连接到服务器

**方式一：双击连接**（推荐）

1. 双击服务器项 "My First Server"
2. EasySSH Lite 将自动唤起系统终端
3. 终端中执行 `ssh` 命令建立连接

**方式二：点击连接按钮**

1. 点击服务器右侧的 **[▶]** 按钮
2. 同样会唤起终端并建立连接

**连接成功后的终端输出：**

```bash

 * Support:        https://ubuntu.com/pro

root@my-server:~#
```

## 常见问题

### Q1: 连接超时/失败

**检查清单：**

```
□ 服务器IP地址是否正确
□ 服务器是否开机且网络可达（ping测试）
□ SSH端口是否正确（默认22）
□ 用户名是否正确
□ 密码是否正确（注意大小写）
□ 服务器防火墙是否允许SSH连接
```

**测试命令：**
```bash
# 测试网络连通性
ping 192.168.1.100

# 测试SSH端口
nc -zv 192.168.1.100 22

# 直接命令行测试
ssh root@192.168.1.100
```

### Q2: 唤起终端失败

**Windows:**
- 确保已安装 Windows Terminal 或 PowerShell
- 如果手动安装了其他终端（如 Tabby、FluentTerminal），EasySSH Lite 会自动检测

**macOS:**
- 确保 Terminal.app 或 iTerm2 已安装
- 检查系统权限：设置 > 安全性与隐私 > 自动化

**Linux:**
- 确保已安装 GNOME Terminal、Konsole 或 XFCE Terminal
- 根据桌面环境自动选择对应终端

### Q3: 密码保存失败

**可能原因：**
- 系统 Keychain/钥匙串被锁定
- 首次使用需要解锁钥匙串

**解决方案：**
```
macOS: 打开 "钥匙串访问"，点击左侧 "登录" 并输入密码解锁
Windows: 检查 Windows 凭据管理器是否正常
Linux: 确保 gnome-keyring 或 kwallet 正在运行
```
