# 教程 02：使用分组管理

> 预计时间：5分钟  |  难度：入门

## 目标

完成本教程后，你将学会：
- 创建和管理服务器分组
- 将服务器移动到不同分组
- 使用分组快速筛选服务器
- 重命名和删除分组

---

## 为什么需要分组？

当你的服务器数量增加时，分组能帮你：

| 场景 | 分组建议 |
|------|----------|
| 生产环境 + 测试环境 | Production / Staging / Development |
| 不同项目 | Project-A / Project-B |
| 不同客户 | Client-张三 / Client-李四 |
| 不同地区 | 北京 / 上海 / 海外 |
| 不同用途 | Web / Database / Cache / Monitor |

## 步骤详解

### 步骤 1：创建分组

**方法一：从设置菜单创建**

1. 点击底部 **[⚙ 设置]** 按钮
2. 选择 **"分组管理"** 标签页
3. 点击 **"[+ 新建分组]"**

```
┌───────────────────────────────────────────────────────┐
│  设置                                                  │
├─────────────────┬─────────────────────────────────────┤
│  通用           │  分组管理                            │
│  外观           │                                      │
│  安全           │  ┌───────────────────────────────┐   │
│  快捷键         │  │  分组名称        服务器数量   │   │
│  分组管理    ←  │  ├───────────────────────────────┤   │
│                 │  │  Ungrouped           6       │   │
│                 │  ├───────────────────────────────┤   │
│                 │  │  [+ 新建分组]                  │   │
│                 │  └───────────────────────────────┘   │
│                 │                                      │
└─────────────────┴─────────────────────────────────────┘
```

4. 输入分组名称 **"Production"**
5. 点击 **"确定"**

**方法二：添加服务器时创建**

在添加服务器的最后一步，选择 **"创建新分组"** 选项。

---

### 步骤 2：移动服务器到分组

**方法一：拖拽移动**（推荐）

```
操作步骤：
1. 按住鼠标左键选中服务器（如 web-prod-1）
2. 拖拽到目标分组标题上
3. 松开鼠标，服务器自动移入该分组

视觉反馈：
- 拖拽时服务器项半透明
- 目标分组标题高亮显示
- 完成后自动重新排序
```

**方法二：右键菜单移动**

```
1. 右键点击服务器项
   ┌───────────────────────────┐
   │  编辑                      │
   │  删除                      │
   │  ─────────────            │
   │  移动到分组      ▶        │
   │    ├─ Production           │
   │    ├─ Staging              │
   │    ├─ Development          │
   │    └─ 创建新分组...       │
   │  ─────────────            │
   │  复制连接命令              │
   └───────────────────────────┘
2. 悬停 "移动到分组" 显示子菜单
3. 点击目标分组名称
```

### 步骤 5：分组展开/折叠

**操作：** 点击分组标题（如 ▼ Production）

```
点击前：
▼ Production (3)           ← 展开状态（显示服务器）
  ├─ 🟢 web-prod-1
  ├─ 🟢 web-prod-2
  └─ 🟢 db-prod

点击后：
▶ Production (3)           ← 折叠状态（隐藏服务器）
```

**快捷键：**
- `Alt + 1~9`：快速展开/折叠第 N 个分组
- `Alt + 0`：展开/折叠所有分组

---

### 步骤 6：重命名分组

**操作步骤：**

1. 在设置中打开 **"分组管理"**
2. 找到要重命名的分组
3. 点击分组名称右侧的 **[✎]** 编辑图标
4. 输入新名称
5. 按 Enter 或点击外部确认

```
编辑状态：
┌──────────────────────────────────┐
│  Production    3    [✎] [🗑]   │
│  [Production Environment____]   │
└──────────────────────────────────┘
```

**注意：**
- Ungrouped 分组无法重命名
- 分组名称不能重复
- 名称长度限制：30 个字符

---

### 步骤 7：删除分组

**操作步骤：**

1. 在 **"分组管理"** 设置中找到分组
2. 点击右侧 **[🗑]** 删除图标
3. 确认对话框：

```
┌─────────────────────────────────────────────┐
│  确认删除                                  │
├─────────────────────────────────────────────┤
│                                             │
│  确定要删除分组 "Staging" 吗？            │
│                                             │
│  ⚠ 该分组内的 2 台服务器将移动到            │
│    "Ungrouped" 分组。                      │
│                                             │
│  [取消]                    [删除]          │
└─────────────────────────────────────────────┘
```

**重要提示：**
- 删除分组不会删除服务器
- 服务器自动移动到 Ungrouped
- 删除前请确认没有重要服务器

---

### 步骤 8：快速筛选分组

**使用搜索栏筛选：**

```
在搜索栏输入分组名称前缀：

🔍 pro

结果：
▼ Production (3)
  ├─ 🟢 web-prod-1    admin@10.0.1.10 ▶
  ├─ 🟢 web-prod-2    admin@10.0.1.11 ▶
  └─ 🟢 db-prod       dba@10.0.1.20   ▶

（其他分组自动隐藏）
```

**高级搜索语法：**

| 搜索语法 | 效果 |
|----------|------|
| `group:prod` | 仅显示 Production 分组 |
| `host:10.0.1` | 显示 IP 包含 10.0.1 的服务器 |
| `user:root` | 显示用户名为 root 的服务器 |
| `status:offline` | 显示离线状态的服务器 |

## 最佳实践

### 推荐分组策略

**方案一：按环境划分**（推荐）
```
Production      - 生产环境，谨慎操作
Staging         - 预发布环境
Development     - 开发测试环境
```

**方案二：按项目划分**
```
E-commerce      - 电商项目
Blog-System     - 博客系统
Internal-Tools  - 内部工具
```

**方案三：按用途划分**
```
Web-Servers     - Web 服务器
Databases       - 数据库服务器
Monitoring      - 监控服务器
CI-CD           - 构建部署服务器
```

### 分组命名规范

| ✅ 推荐 | ❌ 避免 |
|--------|--------|
| 使用英文，简洁清晰 | 使用中文字符（部分终端可能乱码） |
| Production | 生产环境服务器组 |
| Web-Primary | Web服务器第一组 |
| 使用连字符分隔 | 使用空格（搜索时不便） |
| 3-15个字符 | 过长导致界面拥挤 |

## 常见问题

### Q1: 分组太多如何快速找到？

**解决方案：**
1. 使用搜索栏输入 `group:分组名`
2. 折叠其他不常用分组
3. 重新组织分组层级（按环境 > 按项目）

### Q2: 拖拽移动时服务器消失？

**排查步骤：**
```
1. 检查是否拖拽到了正确区域
2. 确保目标分组没有满（上限 100 台/组）
3. 重新启动应用再试一次
4. 改用右键菜单移动
```

### Q3: 如何导出分组配置？

**当前版本：**
- 分组配置随应用配置一起备份
- 使用设置中的 **"导出配置"** 功能

**配置文件位置：**
```
Windows: %APPDATA%\easyssh-lite\groups.json
macOS:   ~/Library/Application Support/easyssh-lite/groups.json
Linux:   ~/.config/easyssh-lite/groups.json
```
