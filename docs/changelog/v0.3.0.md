# EasySSH v0.3.0 Release Notes

**Release Date**: 2026-04-01
**Release Type**: Major Release
**Editions**: Lite, Standard, Pro

---

## Highlights

This is a major release introducing a three-tier product architecture and complete native desktop implementations for all platforms. EasySSH now offers distinct editions tailored to different user needs: Lite for secure configuration storage, Standard for full-featured SSH management, and Pro for team collaboration.

---

## Core Features

### SSH Connection Management

- **Connection Pooling**: Efficient SSH session management with automatic connection reuse
- **Multi-Authentication Support**: Password and SSH key-based authentication
- **Server Metadata**: Comprehensive server information storage including host, port, username
- **Connection Health Monitoring**: Real-time connection status tracking
- **Session Management**: Create, update, delete, and organize SSH connections

### Server Grouping

- **Lite Edition**: Single-level grouping for basic organization
- **Standard Edition**: Nested/hierarchical grouping for complex server landscapes
- **Pro Edition**: Team-based grouping with shared server collections
- **Color Coding**: Visual identification with customizable group colors
- **Search and Filter**: Quick server discovery across all groups

### Credential Storage

- **Encrypted Credentials**: All credentials stored with AES-256-GCM encryption
- **Multiple Auth Methods**: Support for password and SSH key credentials
- **Secure Memory Handling**: Sensitive data automatically cleared from memory (Zeroize)
- **Credential Serialization**: Safe import/export of encrypted credentials

## Security

### Argon2id Key Derivation

- High-security parameters: 64MB memory cost (65536 KB)
- 3 iterations for optimal security/performance balance
- 4 lanes parallelism matching typical CPU cores
- 256-bit output key length
- Protection against GPU/ASIC attacks

### AES-256-GCM Encryption

- NIST-recommended 96-bit nonce length
- Authenticated encryption with associated data (AEAD)
- Unique random nonce per encryption operation
- 256-bit key strength for maximum security

### Keychain Integration

- System keychain integration for secure key storage
- Keychain service identifier: `com.easyssh.lite.crypto`
- Automatic keychain fallback for cross-platform compatibility
- Secure key retrieval with proper error handling

### Additional Security Enhancements

- **RwLock Migration**: Replaced Mutex with RwLock for better concurrent read access
- **Zeroize Implementation**: Automatic memory clearing for sensitive data
- **SSO Security**: PKCE verifier server-side storage, encrypted tokens
- **State Parameter Validation**: High-entropy random strings (32+ characters)
- **Nonce Verification**: Per-session unique nonce to prevent replay attacks
- **Input Validation**: Complete input sanitization and path traversal prevention

## Database

### SQLite Schema

```sql
-- Groups table
CREATE TABLE groups (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL UNIQUE,
    color TEXT NOT NULL DEFAULT '#4A90D9',
    created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

-- Servers table
CREATE TABLE servers (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    host TEXT NOT NULL,
    port INTEGER NOT NULL DEFAULT 22,
    username TEXT NOT NULL,
    auth_method TEXT NOT NULL,
    encrypted_credentials BLOB NOT NULL,
    group_id TEXT,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (group_id) REFERENCES groups(id) ON DELETE SET NULL
);

-- App config table
CREATE TABLE app_config (
    key TEXT PRIMARY KEY,
    value TEXT NOT NULL
);

-- Schema migrations table
CREATE TABLE schema_migrations (
    version INTEGER PRIMARY KEY,
    description TEXT NOT NULL,
    applied_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

### Migration System

- Automatic schema version tracking
- Incremental migration application
- Transaction-safe migration execution
- Migration rollback support for testing
- Version status reporting

### Backup/Restore

- Full database backup with encryption
- Incremental backup support
- Multiple compression formats: gzip, bzip2, zstd, xz
- Cloud storage integration: AWS S3, GCP, Azure
- Reed-Solomon erasure coding for backup integrity
- Scheduled backup with cron-based automation

## Known Issues

### Platform-Specific

1. **Linux GTK4**: Requires pkg-config and GTK4 development libraries
2. **macOS SwiftUI**: Requires Xcode 15+ for SwiftUI features
3. **Windows**: Some antivirus software may flag unsigned executables

### Functional

1. **Keychain Tests**: 5 tests require system keyring access and are skipped in CI
2. **SSH Integration**: Real SSH connections require mock server for testing
3. **SSO Providers**: OIDC/SAML providers must be configured separately

### Performance

1. **Large Server Lists**: Performance may degrade with > 1000 servers (optimization planned)
2. **Backup Size**: Large backups may take significant time to compress
3. **Search Index**: Initial search index build may be slow on first run

## Migration Guide

### Upgrading from 0.2.x to 0.3.x

1. **Backup Database**
   ```bash
   cp ~/.easyssh/easyssh.db ~/.easyssh/easyssh.db.backup
   ```

2. **Install New Version**
   ```bash
   cargo install easyssh-core --version 0.3.0
   ```

3. **Run Migration**
   ```bash
   easyssh migrate
   ```

4. **Verify Installation**
   ```bash
   easyssh --version
   # Expected: EasySSH 0.3.0 (Lite/Standard/Pro)
   ```

### Breaking Changes

1. **Database Schema**: New `schema_migrations` table added
2. **API Changes**: `CryptoState::new()` requires explicit initialization
3. **Configuration Format**: Updated structure for multi-edition support
