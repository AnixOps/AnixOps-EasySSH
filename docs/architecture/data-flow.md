### 1.2 数据流分类

| 类型 | 描述 | 示例 | 安全要求 |
|------|------|------|----------|
| **用户输入流** | 用户操作产生的数据 | 添加服务器、编辑配置 | 即时加密敏感字段 |
| **状态变更流** | 应用状态更新 | 连接状态、会话数据 | 内存安全 |
| **持久化流** | 数据存储到磁盘 | SQLite写入、配置保存 | 加密存储 |
| **网络流** | 与外部系统通信 | SSH连接、Pro同步 | TLS/加密 |
| **审计流** | 操作记录 | 日志写入、会话录制 | 防篡改 |

### 3.2 数据库Schema设计

```sql
-- =============================================
-- EasySSH 数据库 Schema
-- SQLite (Local) + PostgreSQL (Pro Cloud)
-- =============================================

-- 服务器表
CREATE TABLE servers (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    host TEXT NOT NULL,
    port INTEGER DEFAULT 22,
    username TEXT NOT NULL,
    auth_type TEXT NOT NULL CHECK (auth_type IN ('password', 'key', 'agent')),
    auth_encrypted BLOB,  -- 加密存储的密码或私钥路径
    group_id TEXT REFERENCES groups(id) ON DELETE SET NULL,
    tags TEXT,  -- JSON数组
    notes TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 分组表
CREATE TABLE groups (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    parent_id TEXT REFERENCES groups(id) ON DELETE CASCADE,
    color TEXT,  -- hex color
    icon TEXT,
    sort_order INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 连接历史表
CREATE TABLE connection_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    server_id TEXT REFERENCES servers(id) ON DELETE CASCADE,
    connected_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    disconnected_at TIMESTAMP,
    duration_seconds INTEGER,
    exit_code INTEGER,
    terminal_type TEXT
);

-- 命令历史表 (Standard/Pro)
CREATE TABLE command_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    server_id TEXT REFERENCES servers(id) ON DELETE CASCADE,
    session_id TEXT,  -- 关联到connection_history
    command TEXT NOT NULL,
    executed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    exit_code INTEGER,
    output_preview TEXT  -- 前200字符
);

-- Pro版本: 团队表
CREATE TABLE teams (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    owner_id TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    billing_plan TEXT,
    max_members INTEGER DEFAULT 5
);

-- Pro版本: 团队成员
CREATE TABLE team_members (
    id TEXT PRIMARY KEY,
    team_id TEXT REFERENCES teams(id) ON DELETE CASCADE,
    user_id TEXT NOT NULL,
    email TEXT NOT NULL,
    role TEXT NOT NULL CHECK (role IN ('owner', 'admin', 'operator', 'viewer')),
    invited_at TIMESTAMP,
    joined_at TIMESTAMP,
    status TEXT CHECK (status IN ('pending', 'active', 'suspended'))
);

-- Pro版本: 审计日志
CREATE TABLE audit_logs (
    id TEXT PRIMARY KEY,
    team_id TEXT REFERENCES teams(id) ON DELETE CASCADE,
    user_id TEXT NOT NULL,
    event_type TEXT NOT NULL,
    resource_type TEXT,
    resource_id TEXT,
    details JSON,
    ip_address TEXT,
    user_agent TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 索引
CREATE INDEX idx_servers_group ON servers(group_id);
CREATE INDEX idx_servers_host ON servers(host);
CREATE INDEX idx_history_server ON connection_history(server_id);
CREATE INDEX idx_history_time ON connection_history(connected_at);
CREATE INDEX idx_audit_team_time ON audit_logs(team_id, created_at);
```

### A. 数据流状态码

| 状态码 | 含义 | 处理建议 |
|--------|------|----------|
| SYNC_OK | 同步成功 | 无需处理 |
| SYNC_CONFLICT | 数据冲突 | 触发合并流程 |
| SYNC_OFFLINE | 离线状态 | 本地暂存，稍后重试 |
| SYNC_AUTH_FAIL | 认证失败 | 重新登录 |
| SYNC_RATE_LIMIT | 速率限制 | 指数退避重试 |

### B. 性能指标

| 数据流 | 目标延迟 | 最大吞吐量 |
|--------|----------|------------|
| 本地读写 | < 10ms | 10,000 ops/s |
| SSH连接建立 | < 2s | - |
| 终端输入响应 | < 16ms | - |
| 同步推送 | < 500ms | 100 KB/s |
| 监控数据刷新 | < 1s | - |
