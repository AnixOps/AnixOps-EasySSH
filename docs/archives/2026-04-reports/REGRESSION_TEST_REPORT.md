## 发现的问题

### 1. 高优先级

**API Tester WebSocket 测试失败**
- 位置: `api-tester/api-core/src/websocket.rs:165`
- 问题: `test_extract_host` 断言失败
- 预期: `"example.com:8080"`
- 实际: `"example.com"`
- 影响: 低 (仅测试逻辑问题，非功能问题)
- 建议: 检查 `extract_host` 函数端口处理逻辑或更新测试期望

### 2. 中优先级

**备份功能编译错误**
- 位置: `core/src/backup/` 模块
- 问题: Uuid serde trait 未满足
- 错误: `the trait bound Uuid: serde::Serialize is not satisfied`
- 影响: 中 (backup 功能不可用)
- 建议: 检查 Cargo.toml 中 uuid 依赖的 serde 特性配置

**AWS SDK 版本不兼容**
- 位置: 所有 AWS 相关功能
- 问题: 需要 Rust 1.91.1，当前 1.89.0
- 影响: 中 (cloud-aws 功能不可用)
- 建议: 升级 Rust 工具链或降级 AWS SDK 版本

### 3. 低优先级

**未使用变量警告**
- 数量: 约 60 个警告
- 影响: 低 (代码质量)
- 建议: 清理未使用变量或添加 `_` 前缀

**Pro-server 测试占位符**
- 位置: `pro-server/tests/integration_tests.rs`
- 问题: 10 个测试为占位符，未实际执行
- 影响: 低 (需要后续实现)
