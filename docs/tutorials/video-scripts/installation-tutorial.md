### 【macOS 安装】(2:00 - 3:15)

**[章节标题卡]**
"macOS 安装"

**[画面]**
1. **下载** (2:00 - 2:15)
   - Safari 访问下载页面
   - 点击 "macOS 下载"
   - 下载 DMG 文件

2. **安装** (2:15 - 2:45)
   - 下载完成后自动打开 DMG
   - 拖拽应用到 Applications 文件夹
   - 进度条显示复制过程

3. **安全验证** (2:45 - 3:15)
   - 首次启动时的安全提示
   - 显示"无法打开，因为无法验证开发者"
   - 演示如何允许：
     - 系统设置 → 隐私与安全性
     - 安全性部分："已阻止使用 EasySSH Lite"
     - 点击"仍要打开"
   - 再次启动应用
   - 主界面展示

**[旁白]**
"macOS 安装采用 DMG 拖拽方式，这是苹果用户熟悉的方法。下载完成后，只需将 EasySSH Lite 图标拖拽到 Applications 文件夹。"

"首次启动时，macOS 会提示无法验证开发者。这是因为我们是新开发者，正在申请 Apple Developer ID。要允许运行，打开系统设置，进入隐私与安全性，在安全性部分点击'仍要打开'。"

"这是标准的安全流程，只需要设置一次，之后就可以正常启动。"

**[字幕]**
- "拖拽到 Applications 文件夹"

### 【Linux 安装】(3:15 - 4:15)

**[章节标题卡]**
"Linux 安装"

**[画面]**
1. **多种安装方式** (3:15 - 3:30)
   - 显示三种安装选项：AppImage, deb, rpm
   - 说明不同发行版的选择

2. **AppImage 方式** (3:30 - 3:55)
   - 下载 `.AppImage` 文件
   - 终端命令：
     ```bash
     chmod +x EasySSH-Lite-*.AppImage
     ./EasySSH-Lite-*.AppImage
     ```
   - 应用直接启动

3. **Deb 包方式** (3:55 - 4:15)
   - Ubuntu/Debian 示例
   - 终端命令：
     ```bash
     sudo dpkg -i easyssh-lite_1.0.0_amd64.deb
     sudo apt-get install -f  # 修复依赖
     ```
   - 从应用菜单启动

**[旁白]**
"Linux 用户有三种安装选择。最简单的是 AppImage，它不需要安装，下载后赋予执行权限即可运行。"

"对于 Ubuntu 和 Debian 用户，我们提供了 deb 包。使用 dpkg 命令安装，如果有依赖问题，运行 apt-get install -f 自动修复。安装后，EasySSH Lite 会出现在你的应用菜单中。"

"Fedora 和 openSUSE 用户可以选择 rpm 包，安装方式类似。"

**[字幕]**
- "AppImage: 无需安装，双击运行"
- "deb/rpm: 系统包管理器安装"
- "官网下载页面有详细说明"
