# EasySSH Standard 版本规划

> 全功能跨平台SSH客户端

---

## 1. 版本定位

### 核心价值
- **完整功能**：内置终端 + 分屏 + 监控
- **高性能**：WebGL加速，60fps渲染
- **可扩展**：未来可平滑升级到Pro

### 目标用户
- 多服务器管理者（10-100台）
- 需要分屏对比日志的运维
- 需要监控服务器状态的开发者

### 成功指标
- 安装包 20-30MB
- 终端输入延迟 < 16ms
- 60fps 渲染性能
- 支持100+并发标签

### SSH连接复用

```rust
// 在单个SSH会话中创建多个channel
// 避免为每个操作建立新连接
struct SshSession {
    session: Arc<Mutex<Session>>,

    // 前台channel (交互式终端)
    interactive: Option<Channel>,

    // 后台channels (监控/命令执行)
    background: Vec<Channel>,
}

impl SshSession {
    // 复用连接执行命令
    pub async fn exec(&self, cmd: &str) -> Result<String> {
        let mut channel = self.session.lock().await.channel_session()?;
        channel.exec(cmd)?;
        // ... 读取输出
    }

    // 监控数据使用独立channel，不影响交互
    pub async fn monitor(&self, metric: &str) -> Result<String> {
        let mut channel = self.session.lock().await.channel_session()?;
        channel.exec(&format!("cat /proc/{}", metric))?;
        // ... 读取输出
    }
}
```

## 5. 性能目标

| 指标 | 目标 | 测量方法 |
|------|------|----------|
| 冷启动时间 | < 3秒 | 进程启动到首帧渲染 |
| 终端输入延迟 | < 16ms | 击键到屏幕显示 |
| 渲染帧率 | 60fps | 高频输出时 |
| 内存占用 | < 200MB | 10个标签页 |
| CPU占用(空闲) | < 1% | 无活动时 |
