# v0.3.0 Beta 规划文档

> 第一个公开Beta版本规划 - EasySSH三版本产品战略

---

## 版本概述

**版本号**: v0.3.0-beta
**代号**: Foundation
**定位**: 第一个公开Beta版本
**目标用户**: 早期测试用户、技术爱好者、企业预览用户

### 核心价值主张

| 版本 | Beta 目标 |
|------|-----------|
| **Lite** | 验证SSH配置管理市场，证明"配置保险箱"概念可行性 |
| **Standard** | 验证嵌入式终端 + 分屏架构，收集开发者工作流反馈 |
| **Pro** | 验证团队管理 + 审计功能，确认企业需求匹配度 |

## 质量门禁

### 准入标准 (发布前必须满足)

| 指标 | 目标 | 测量方法 |
|------|------|----------|
| 测试覆盖率 | > 75% | `cargo tarpaulin` / `jest --coverage` |
| 安全漏洞 | 零高危 | `cargo audit` + 手动审计 |
| 编译错误 | 零错误 | Windows/Linux CI全绿 |
| 启动性能 | < 2秒 | 冷启动时间测量 |
| 内存占用 | < 150MB | 基准测试 |
| 崩溃率 | < 0.1% | 内部测试统计 |

### 各版本具体目标

**Lite版本**:
- 安装包大小 < 15MB
- 冷启动 < 1.5秒
- 内存占用 < 80MB (空闲)

**Standard版本**:
- 安装包大小 < 50MB
- 冷启动 < 2秒
- 单终端标签内存 < 20MB

**Pro版本**:
- 离线模式完全可用
- 云端同步延迟 < 5秒
- API响应时间 < 500ms (P95)

- 极少数SSH服务器握手兼容性问题

## 附录

### 版本命名约定

- **Alpha**: 内部测试，功能不完整
- **Beta**: 公开测试，功能完整，可能有Bug
- **RC** (Release Candidate): 候选版本，准备正式 release
- **Stable**: 正式稳定版本

### Beta用户协议要点

1. 数据可能丢失 (需备份)
2. 功能可能变更
3. 自动匿名遥测收集
4. 鼓励反馈但不保证响应时效
