# 教程 04：导入 SSH 配置

> 预计时间：5分钟  |  难度：入门

## 目标

完成本教程后，你将学会：
- 理解 OpenSSH 配置文件格式
- 从 `~/.ssh/config` 批量导入服务器
- 处理包含高级选项的配置
- 验证导入结果
- 定期同步配置

## 为什么使用配置导入？

### 手动添加 vs 批量导入

| 场景 | 方法 | 时间 |
|------|------|------|
| 新增 1-2 台服务器 | 手动添加 | 2分钟 |
| 已有 10+ 台服务器配置 | 批量导入 | 30秒 |
| 团队协作共享配置 | 导入+导出 | 1分钟 |

**典型用户场景：**
```
你已经维护 ~/.ssh/config 多年：
- 10+ 台生产服务器
- 5 台测试服务器
- 3 个不同账户
- 各种 ProxyJump 配置

不想一台台重新录入 EasySSH Lite？
一键导入解决！
```

## OpenSSH 配置文件格式

### 基本结构

```bash
# ~/.ssh/config

# 第一个服务器配置
Host web-server
    HostName 192.168.1.10
    User admin
    Port 22
    IdentityFile ~/.ssh/id_rsa

# 第二个服务器配置
Host db-server
    HostName 192.168.1.20
    User dba
    Port 2222
```

### 支持的配置选项

| SSH 选项 | EasySSH Lite 支持 | 说明 |
|----------|-------------------|------|
| `Host` | 作为服务器名称 | 必填 |
| `HostName` | 作为主机地址 | 必填 |
| `User` | 作为用户名 | 必填 |
| `Port` | 作为端口 | 默认 22 |
| `IdentityFile` | 作为密钥路径 | 认证方式 |
| `PasswordAuthentication` | 转换为密码认证 | 是/否 |
| `ProxyJump` | 暂不支持 | 标记为备注 |
| `LocalForward` | 暂不支持 | 标记为备注 |
| `ServerAliveInterval` | 暂不支持 | 忽略 |

### 复杂配置示例

```bash
# ~/.ssh/config

# ======================================
# 生产环境
# ======================================
Host prod-web-1
    HostName 10.0.1.10
    User deploy
    Port 22
    IdentityFile ~/.ssh/prod_key
    ServerAliveInterval 60

Host prod-web-2
    HostName 10.0.1.11
    User deploy
    IdentityFile ~/.ssh/prod_key

Host prod-db
    HostName 10.0.1.20
    User dba
    Port 2222
    IdentityFile ~/.ssh/prod_key

# ======================================
# 通过跳板机连接
# ======================================
Host bastion
    HostName 203.0.113.10
    User admin
    IdentityFile ~/.ssh/bastion_key

Host internal-server
    HostName 192.168.100.5
    User developer
    ProxyJump bastion
    IdentityFile ~/.ssh/internal_key

# ======================================
# 测试环境（密码认证）
# ======================================
Host test-server
    HostName 10.0.2.10
    User test
    PasswordAuthentication yes
    # 注意：EasySSH Lite 会提示输入密码
```

## 步骤详解

### 步骤 1：查看现有 SSH 配置

**找到配置文件：**

```bash
# 配置文件路径
~/.ssh/config          # Linux/macOS
C:\Users\用户名\.ssh\config    # Windows

# 查看内容
cat ~/.ssh/config
```

**配置文件示例：**

```
# ~/.ssh/config

Host my-server
    HostName 192.168.1.100
    User root
    Port 22
    IdentityFile ~/.ssh/id_rsa

Host web-prod
    HostName 203.0.113.50
    User ubuntu
    IdentityFile ~/.ssh/prod_key

Host db-staging
    HostName 10.0.0.20
    User dba
    Port 2222
```

## 高级用法

### 定期同步配置

**场景：** 团队成员共享 SSH 配置

```bash
# 团队共享配置仓库
git clone https://company.com/team-ssh-configs.git

# 每次更新后重新导入 EasySSH Lite
cd team-ssh-configs
git pull
# 然后在 EasySSH Lite 中重新导入 config 文件
```

### 导入特定配置块

**使用注释标记：**

```bash
# ~/.ssh/config

# === EasySSH: Production ===
Host prod-web-1
    HostName 10.0.1.10
    User deploy

Host prod-db
    HostName 10.0.1.20
    User dba
# === End EasySSH ===

# 以下配置不会被导入（个人服务器）
Host my-home-server
    HostName 192.168.0.50
    User admin
```

EasySSH Lite 只会导入标记块内的配置。

### 批量导入后编辑

## 常见问题

### Q1: 找不到 ~/.ssh/config 文件

**检查步骤：**

```bash
# 1. 检查 .ssh 目录是否存在
ls -la ~/
# 如果不存在，创建
mkdir ~/.ssh
chmod 700 ~/.ssh

# 2. 创建空的 config 文件
touch ~/.ssh/config
chmod 600 ~/.ssh/config

# 3. 现在可以导入了（虽然为空）
```

### Q2: 导入后密钥认证失败

**排查步骤：**

```bash
# 1. 检查密钥文件路径是否正确
ls -la ~/.ssh/id_rsa

# 2. 检查密钥权限
chmod 600 ~/.ssh/id_rsa

# 3. 如果密钥有密码短语，在 EasySSH Lite 中重新输入

# 4. 测试命令行是否能用密钥连接
ssh -i ~/.ssh/id_rsa user@hostname
```

### Q3: ProxyJump/跳板机配置如何处理？

**当前版本：**
- ProxyJump 配置会被保存为服务器备注
- 连接时不会自动使用跳板机

**临时解决方案：**

```bash
# 方法 1：使用 SSH Agent 和 ProxyCommand
# 在终端手动配置

# 方法 2：先在 EasySSH Lite 中连接跳板机
# 然后在跳板机上连接目标服务器

# 方法 3：等待未来版本支持（Pro 版将支持）
```

### Q4: 导入后出现重复服务器

**原因：** 配置文件中有别名或多重 Host 定义

```bash
# 例如：
Host web-server web-server-alias
    HostName 192.168.1.10
    User admin
```

**解决方案：**
- EasySSH Lite 会自动检测重复（相同 HostName + User）
- 提示用户选择：跳过 / 更新 / 重命名导入

### Q5: 如何导出 EasySSH Lite 配置到 SSH config？

**反向操作：**

1. 设置 → 导入/导出 → **"[导出到 SSH Config]"**
2. 选择要导出的服务器
3. 选择导出格式（简洁/详细）
4. 保存文件或复制到剪贴板

```
导出格式选项：
○ 简洁格式（仅包含必要字段）
● 详细格式（包含所有支持的字段）
○ 分组注释（添加分组标记）
```
