## 使用场景推荐

### 选择 Lite 版

**适合用户：**
- 个人开发者
- 注重隐私，希望数据完全本地存储
- 习惯使用系统原生终端（iTerm2、Windows Terminal、Alacritty 等）
- 不需要高级终端功能

**典型场景：**
```
"我有 10 台云服务器需要管理。
我习惯用 iTerm2，只想安全地保存连接配置。
我不想学复杂的界面操作。"
```

### 选择 Standard 版

**适合用户：**
- 运维工程师
- 全栈开发者
- 需要同时管理多台服务器
- 需要文件传输功能

**典型场景：**
```
"我每天要连接 20+ 台服务器。
需要在不同会话间快速切换。
经常需要上传下载文件。
想要分屏同时监控多台服务器。"
```

### 选择 Pro 版

**适合用户：**
- 开发团队 Leader
- DevOps / SRE 团队
- 企业 IT 管理员
- 需要合规审计的组织

**典型场景：**
```
"我们团队有 50 名工程师，需要共享服务器配置。
需要知道谁在什么时候连接了生产环境。
新员工入职时需要快速获得访问权限。
需要通过 SSO 统一身份认证。
需要满足 SOC2 合规要求。"
```

## 升级路径

### Lite → Standard

**无缝升级：**
1. 安装 Standard 版
2. 自动读取 Lite 的数据
3. 配置和服务器信息完整保留
4. 获得嵌入式终端等新功能

### Standard → Pro

**需要服务端：**
1. 部署 Pro 服务端（详见部署文档）
2. 在 Standard 客户端中配置服务端地址
3. 可选择迁移本地数据到团队空间
4. 获得团队协作功能

### 降级

**Pro → Standard/Lite:**
- 导出数据为通用格式
- 重新安装目标版本
- 导入数据

**Standard → Lite:**
- 仅保留服务器配置
- 终端会话历史、分屏布局等将丢失

## 常见问题

**Q: 可以在同一设备安装多个版本吗？**
A: 不建议。Lite 和 Standard 使用相同的数据目录，会相互影响。

**Q: 团队使用 Lite/Standard 版可以吗？**
A: 技术上可以，但无法共享配置和审计。Pro 版专为团队设计。

**Q: Pro 版可以离线使用吗？**
A: 有限支持。已缓存的数据可离线访问，但同步和团队功能需要网络。

**Q: 可以混合使用不同版本吗？**
A: 同一团队建议统一版本。个人可独立使用 Lite/Standard。

**Q: 数据格式兼容吗？**
A: 基础数据（服务器、分组、密钥）完全兼容。Pro 特有功能数据在降级时会丢失。
